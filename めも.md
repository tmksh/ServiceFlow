## 案件コメント（社内チャット）機能 設計メモ

### 概要
案件ごとにスタッフ間でコメント・連絡事項を共有できるチャット機能。
現在はフロントのローカルstate管理のみのモックとして実装済み（`QuickCaseModal`内）。

### DBテーブル設計（Supabase / MySQL）

```sql
-- 案件コメントテーブル
CREATE TABLE case_comments (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  case_id     VARCHAR(20) NOT NULL,         -- 案件ID (例: CS-0001)
  user_id     UUID NOT NULL,                -- 投稿者
  body        TEXT NOT NULL,                -- コメント本文
  created_at  TIMESTAMP DEFAULT now(),
  updated_at  TIMESTAMP DEFAULT now(),
  deleted_at  TIMESTAMP                     -- 論理削除
);

-- インデックス
CREATE INDEX idx_case_comments_case_id ON case_comments(case_id);
CREATE INDEX idx_case_comments_created_at ON case_comments(created_at);
```

### リアルタイム同期（Supabase Realtime）

```typescript
// 案件コメントをリアルタイム購読
const channel = supabase
  .channel(`case-comments:${caseId}`)
  .on('postgres_changes', {
    event: 'INSERT',
    schema: 'public',
    table: 'case_comments',
    filter: `case_id=eq.${caseId}`,
  }, (payload) => {
    setComments((prev) => [...prev, payload.new as Comment]);
  })
  .subscribe();
```
