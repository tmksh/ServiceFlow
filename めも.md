# スマカレ 設計メモ

## 背景・課題

- LINEグループへのコールセンター投稿を自動でシステムに取り込みたい
- 電話をシステムから発信・録音して営業分析に活用したい
- LINEで来たメッセージをコピーしてTimeTreeに入力するのが面倒 → 自動化したい

## 案件コメント（社内チャット）機能 設計メモ

### 概要

案件ごとにスタッフ間でコメント・連絡事項を共有できるリアルタイムチャット機能。

**現状**: フロントのローカルstate管理のみのモックとして実装済み（`src/components/ui/quick-case-modal.tsx` 内）。  
**Phase 3**: Supabase Realtime に接続して本番動作させる。

### DBテーブル設計

```sql
-- 案件コメントテーブル
CREATE TABLE case_comments (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  case_id     TEXT NOT NULL REFERENCES cases(id),
  user_id     UUID NOT NULL REFERENCES members(id),
  body        TEXT NOT NULL,
  created_at  TIMESTAMPTZ DEFAULT now(),
  updated_at  TIMESTAMPTZ DEFAULT now(),
  deleted_at  TIMESTAMPTZ  -- 論理削除
);

CREATE INDEX idx_case_comments_case_id ON case_comments(case_id);
CREATE INDEX idx_case_comments_created_at ON case_comments(created_at);
```

### リアルタイム同期（Supabase Realtime）

```typescript
// 案件コメントをリアルタイム購読
const channel = supabase
  .channel(`case-comments:${caseId}`)
  .on('postgres_changes', {
    event: 'INSERT',
    schema: 'public',
    table: 'case_comments',
    filter: `case_id=eq.${caseId}`,
  }, (payload) => {
    setComments((prev) => [...prev, payload.new as Comment]);
  })
  .subscribe();
```

### フロント接続ポイント

- `src/components/ui/quick-case-modal.tsx` のコメントセクション
- `Comment` 型・`INITIAL_COMMENTS` を本番APIレスポンスに差し替えるだけで接続可能

## LINE連携 設計メモ

### 概要

LINEグループに投稿されたコールセンタースタッフのテキストを、Webhookで自動受信し案件登録するパイプライン。

### 処理フロー

```
LINEグループ投稿（コールセンタースタッフ）
  ↓
LINE Messaging API（Webhook）
  ↓
POST /api/line/webhook
  1. HMAC-SHA256 署名検証
  2. line-parser.ts でテキスト解析
  3. line_messages テーブルに保存（status: "pending"）
  4. 自動登録モードON → cases テーブルに INSERT（is_line_auto: true）
  5. 「緊急」「当日」「即日」を含む → is_urgent: true
  ↓
Supabase Realtime → フロントに即時反映
```

### 解析対象フォーマット（実データより）

```
神奈川 ネコ 受電 滝沢         ← 1行目: 都道府県 センター チャネル オペレーター
あかざわ たつゆき              ← 顧客名
〒248-0005                    ← 郵便番号
神奈川県逗子市桜山5-39-16      ← 住所
090-6195-0960                 ← 電話番号
2/17 9-12時                   ← 日付 時間帯
不用品 ベッドフレーム・マットレス ← 品目
```

### 必要な環境変数

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=

# LINE
LINE_CHANNEL_SECRET=
LINE_CHANNEL_ACCESS_TOKEN=
```

### 実装ファイル一覧（Phase 3）

| ファイル | 役割 |
|----------|------|
| `src/lib/line-parser.ts` | テキスト解析ロジック（正規表現・行解析） |
| `src/app/api/line/webhook/route.ts` | Webhook受信・署名検証・保存 |
| `src/app/api/line/messages/route.ts` | メッセージ一覧取得 |
| `src/app/api/line/messages/[id]/register/route.ts` | 手動案件登録 |

## 通話AI 設計メモ（Phase 4）

### 概要

スマカレ上から顧客に発信 → 通話録音 → AI文字起こし・要約 → 案件に自動保存。

### 処理フロー

```
スマカレ上で電話番号クリック
  ↓
Twilio Voice API（発信・録音）
  ↓
通話終了 → MP3生成 → S3保存
  ↓
POST /api/twilio/recording（Webhook）
  ↓
OpenAI Whisper API（文字起こし）
  ↓
GPT-4o API（要約・情報抽出）
  ↓
cases テーブルの call_log に自動保存
```

### コスト試算

| API | 単価 | 月間1000件の場合 |
|-----|------|-----------------|
| Twilio（録音込み） | 約¥15/分 | 約¥150,000（10分/件想定） |
| Whisper | ¥0.9/分 | 約¥9,000 |
| GPT-4o | ¥3/件 | 約¥3,000 |
| **合計** | | **約¥162,000/月** |
